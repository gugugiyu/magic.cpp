## Overview

Five independently-toggled built-in tools injected into the model's tool list before each request. Unlike MCP tools (which require an external server), built-in tools execute entirely within the frontend.

| Tool                  | Purpose                                                        |
| --------------------- | -------------------------------------------------------------- |
| `call_subagent`       | Delegate a task to a separate model endpoint                   |
| `calculator`          | Evaluate a math expression (confirm-gated)                     |
| `get_time`            | Return current local date, time, and timezone                  |
| `get_location`        | Return approximate geolocation (browser permission)            |
| `sequential_thinking` | Emit structured reasoning steps; rendered as a live stepper UI |

## 5. Execution in `executeBuiltinTool()` — `src/lib/stores/agentic.svelte.ts`

```typescript
case 'calculator': {
  const { expression } = args as { expression: string };
  const confirmed = window.confirm(`Allow calculator to evaluate:\n\n${expression}`);
  if (!confirmed) return { error: 'User denied calculator execution.' };
  try {
    // eslint-disable-next-line no-eval
    const result = eval(expression);
    return { result: String(result) };
  } catch (e) {
    return { error: `Evaluation error: ${e instanceof Error ? e.message : String(e)}` };
  }
}

case 'get_time': {
  const now = new Date();
  return {
    result: {
      iso:      now.toISOString(),
      local:    now.toLocaleString(),
      timezone: Intl.DateTimeFormat().resolvedOptions().timeZone
    }
  };
}

case 'get_location': {
  return new Promise((resolve) => {
    navigator.geolocation.getCurrentPosition(
      (pos) => resolve({
        result: {
          latitude:  pos.coords.latitude,
          longitude: pos.coords.longitude,
          accuracy:  pos.coords.accuracy
        }
      }),
      (err) => resolve({ error: `Geolocation error: ${err.message}` })
    );
  });
}

case 'sequential_thinking': {
  const { thought, thoughtNumber, totalThoughts, nextThoughtNeeded } =
    args as { thought: string; thoughtNumber: number; totalThoughts: number; nextThoughtNeeded: boolean };

  sequentialThinkingStore.recordThought({
    thought,
    thoughtNumber,
    totalThoughts,
    nextThoughtNeeded,
    conversationId,  // passed from outer context
    turnIndex        // current assistant turn index
  });

  return { result: 'Thought recorded' };
}

case 'call_subagent': {
  // Implementation deferred — see subagent infrastructure section below
  return { error: 'call_subagent not yet implemented.' };
}
```

## 6. Sequential thinking — store

**File:** `src/lib/stores/sequential-thinking.svelte.ts` (new file)

### Data shape

```typescript
export interface ThoughtEntry {
  thoughtNumber:    number;
  totalThoughts:    number;
  thought:          string;
  nextThoughtNeeded: boolean;
  done:             boolean;   // true once nextThoughtNeeded = false OR a later thought arrives
}

export interface ThinkingTurn {
  turnIndex:      number;
  conversationId: string;
  thoughts:       ThoughtEntry[];
}
```

### Store API

```typescript
class SequentialThinkingStore {
  turns = $state<ThinkingTurn[]>([]);

  // Called by executeBuiltinTool on each sequential_thinking call
  recordThought(payload: {
    thought: string;
    thoughtNumber: number;
    totalThoughts: number;
    nextThoughtNeeded: boolean;
    conversationId: string;
    turnIndex: number;
  }): void;

  // Returns the active turn for a given conversationId + turnIndex
  getTurn(conversationId: string, turnIndex: number): ThinkingTurn | undefined;

  // Returns all turns for a given conversationId (for the history drawer)
  getTurnsForConversation(conversationId: string): ThinkingTurn[];

  // Clear all thoughts for a conversation (e.g. on conversation delete)
  clearConversation(conversationId: string): void;
}

export const sequentialThinkingStore = new SequentialThinkingStore();
```

> **Note:** Thoughts live in memory only (not persisted to IndexedDB). They are reconstructed if the conversation is reloaded by re-running the model — this is acceptable since thoughts are ephemeral reasoning, not conversation content.

Step states:

| State       | Icon          | Description                                           |
| ----------- | ------------- | ----------------------------------------------------- |
| `done`      | `✓` (green)   | `nextThoughtNeeded` was false, or a later thought arrived |

## 9. Subagent — infrastructure (surface behavior)

> Implementation details deferred to next session.

The `call_subagent` tool requires a separate backend endpoint (OpenAI-compatible). The connection is configured under **Settings → Connection** in a dedicated "Subagent" sub-group:

- **Enable subagent** toggle (gates the tool registration)
- **Endpoint URL** — base URL of the subagent server
- **API key** — optional, separate from main endpoint
- **Model selector** — populated from `GET {endpoint}/v1/models`
- **Summarize long text** toggle — when on, text exceeding a token threshold is summarized by the subagent before reaching the main model

A secondary badge displaying the active subagent model is shown beside the main `ModelBadge` in `ChatFormTextarea`.

All subagent state lives in `subagentConfigStore` — a dedicated store isolated from `apiConfigStore`.
