## Pipeline insertion point

Filters are **display-only** — applied in the Svelte rendering layer, NOT in the store or DB.

Rendering path for assistant text:

```
chat.svelte.ts  (streamChatCompletion → onChunk/onComplete)
  └→ conversationsStore.updateMessageAtIndex  (stores raw content)
       └→ ChatMessageAssistant.svelte
            └→ ChatMessageAgenticContent.svelte   ← APPLY FILTERS HERE
                 └→ MarkdownContent.svelte  (remark/rehype pipeline)
```

Each `AgenticSectionType.TEXT` segment in `ChatMessageAgenticContent.svelte` is passed
individually to `MarkdownContent`. The filter runs on `section.content` before that call.

Language pinner is the only **pre-request** filter. It runs in
`src/lib/services/chat.service.ts` after `normalizedMessages` is built (line ~131),
before `requestBody` is assembled (line ~133). It modifies only the in-memory
`normalizedMessages` array, never the DB.

## Files to create / modify

### 1. NEW `src/lib/utils/filters.ts`

Four exported functions + one composite:

```typescript
// Strip all Extended_Pictographic emoji sequences (including ZWJ + skin-tone combos)
export function applyEmojiRemoval(text: string): string;

// Keep only the first fenced code block; if none found, return text unchanged
export function applyCodeblockOnly(text: string): string;

// Strip all Markdown syntax, return plain text
export function applyRawMode(text: string): string;

// Return language code from first "![xx]" match, or null
export function detectLanguagePinner(text: string): string | null;

export interface ResponseFilterOptions {
	filterEmojiRemoval: boolean;
	filterCodeblockOnly: boolean;
	filterRawMode: boolean;
}

// Composite: applies enabled filters in order: codeblock → emoji → raw
export function applyResponseFilters(text: string, opts: ResponseFilterOptions): string;
```

Implementation notes:

- `applyEmojiRemoval`: use `/\p{Extended_Pictographic}(?:[\uFE00-\uFE0F\u{1F3FB}-\u{1F3FF}\u{20E3}]|\u{200D}\p{Extended_Pictographic}(?:[\uFE00-\uFE0F\u{1F3FB}-\u{1F3FF}\u{20E3}])?)*` with `gu` flag
- `applyCodeblockOnly`: `/```[\s\S]*?```/` — return first match or original
- `applyRawMode`: sequential `.replace()` chain stripping headings, bold, italic, code, links, lists, blockquotes, hr
- `detectLanguagePinner`: `/!\[([a-zA-Z]{2,8})\]/` — return `match[1]` or null

```typescript
// Language pinner: if enabled, detect ![xx] in the last user message and
// append a language instruction to the API request only (not stored).
if (settingsStore.config.filterLanguagePinner) {
	const lastUserMsg = [...normalizedMessages].reverse().find((m) => m.role === MessageRole.USER);
	if (lastUserMsg) {
		const textContent =
			typeof lastUserMsg.content === 'string'
				? lastUserMsg.content
				: ((lastUserMsg.content as ApiChatMessageContentPart[])?.find(
						(p) => p.type === ContentPartType.TEXT
					)?.text ?? '');
		const lang = detectLanguagePinner(textContent);
		if (lang) {
			const instruction = `\n\nIMPORTANT: the response should be in ${lang}.`;
			if (typeof lastUserMsg.content === 'string') {
				lastUserMsg.content += instruction;
			} else if (Array.isArray(lastUserMsg.content)) {
				const textPart = (lastUserMsg.content as ApiChatMessageContentPart[]).find(
					(p) => p.type === ContentPartType.TEXT
				);
				if (textPart) {
					textPart.text += instruction;
				} else {
					(lastUserMsg.content as ApiChatMessageContentPart[]).push({
						type: ContentPartType.TEXT,
						text: instruction.trim()
					});
				}
			}
		}
	}
}
```

Note: `normalizedMessages` entries are fresh objects (created by `convertDbMessageToApiChatMessageData`),
so mutating them here is safe and never touches the DB.

## Filter application order (display)

When multiple filters are active simultaneously:

1. **Codeblock only** — narrow content to first code block
2. **Emoji removal** — strip emojis from remaining content
3. **Raw mode** — strip Markdown syntax last

---

## Streaming behaviour

Filters are applied purely at render time on `section.content`. Since streaming continuously
updates `section.content` via reactive store updates, the filter re-applies on every chunk.
This means:

- **Emoji removal** visually "hides" emojis mid-stream (before full response) — acceptable
  per spec since it is display-only and the spec only restricts _persisted_ emoji removal to
  post-completion.
- **Codeblock only** will show nothing until the opening ``` appears, then show the partial
  block — acceptable.
- **Raw mode** strips incrementally — acceptable.
- **Language pinner** is a pre-request filter, unaffected by streaming.
